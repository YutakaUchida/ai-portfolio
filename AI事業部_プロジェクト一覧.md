## 2. インバウンドコール分析

**フォルダ名:** Thinca

### 解決する課題
コールセンターの通話録音（通録）データが大量にあるが、これまで分析したことがなかった。カイクラ（通録システム）からの手動ダウンロードも時間がかかり、分析の前段階で止まっていた。

### 概要
カイクラから通話履歴を自動でダウンロードし、AIが音声を解析するツールです。これまで手が付けられなかった大量の通録データを、初めて分析できるようになりました。

### 主な機能
- 通録履歴の自動一括ダウンロード（ページ送りも自動対応）
- AIによる音声の文字起こし・要約
- 顧客の感情分析・キーワード抽出

### 使用技術
Node.js / Playwright / Google Gemini API

### 対象ユーザー
コールセンター品質管理チーム

## 3. Chatwork問い合わせ一元管理

**フォルダ名:** chatwork

### 解決する課題
社内のサポートセンターが1つのChatworkアカウントを共有して使っていたため、誰かが未読メッセージを見ると既読になってしまい、他のスタッフが見落とすことがあった。また、プランナーへのエスカレーション連絡も手作業で手間がかかっていた。

### 概要
Chatworkの複数ルームに届く問い合わせを、スプレッドシートで一元管理できるようにしたツールです。既読/未読に関係なく全メッセージが記録されるため見落としがなくなり、プランナーへのエスカレーションもボタン一つで実行できます。

### 主な機能
- 複数ルームの問い合わせをスプレッドシートに自動記録
- 既読/未読に関係なく全メッセージを確実に管理
- 宛先による自動分類（gnote関連 / WEB関連）
- プランナーへのエスカレーションをボタン一つで実行

### 使用技術
Google Apps Script / Chatwork API / Google Sheets

### 対象ユーザー
サポートセンター

## 4. gnote入庫実績レポート

**フォルダ名:** gnote_report

### 解決する課題
RPAでクライアントごとにCSVをダウンロードし、スプレッドシートで集計、Looker Studioに反映という流れで作業していた。定期的に大量のクライアントの作業をしていたため、RPAの稼働時間を大量に使っていた。また、Looker Studioもクライアントごとに作成していたため約400個も存在し、修正・管理が大変だった。

### 概要
gnoteの入庫実績データをBigQueryで完全自動化し、レポートを1つに集約したシステムです。RPAの稼働時間を大幅に削減。パラメーターで出し分けできるようになったため、約400個あったLooker Studioが1つに。全クライアント分の見せ方を一気に変更したり、新規クライアント対応、店舗の追加・削除などの修正・更新も簡単になりました。

### 主な機能
- RPAによるCSVダウンロード・集計作業を完全自動化
- RPAの稼働時間を大幅削減
- 約400個のレポートを1つに集約（パラメーターで出し分け）
- 全クライアント分の見せ方変更、新規クライアント対応、店舗の追加・削除が簡単に

### 使用技術
Google BigQuery / Looker Studio / Google Apps Script

### 対象ユーザー
経営層、店舗管理者

## 5. gnoteコール結果レポート

**フォルダ名:** gnote_report

### 解決する課題
gnoteから手作業でCSVをダウンロードする必要があった。コール結果レポートはLooker Studioすらなく、スプレッドシートのテーブルを見せているだけだった。店舗別やエリア別などの切り口での分析ができなかった。

### 概要
gnoteのコール結果データをBigQueryで自動化し、自作HTMLで柔軟かつグラフィカルなレポートを実現したシステムです。見栄えが一気に改善され、店舗別・エリア別など今まで見れなかった切り口での分析が可能に。パラメーターで出し分けているので、全クライアントに対して見せ方を変更したり、新しいグラフを追加したりも一気にできます。

### 主な機能
- gnoteからのCSVダウンロード・集計を完全自動化
- 自作HTMLによるグラフィカルなレポート表示
- 店舗別・エリア別など様々な切り口での分析
- 全クライアントへの見せ方変更・新しいグラフ追加も一気に対応

### 使用技術
Google BigQuery / 自作HTML / Google Apps Script

### 対象ユーザー
コールセンターマネージャー

## 7. gnote Direct（画面操作でデータ抽出）

**フォルダ名:** gnote_report

### 解決する課題
CSVを見るだけやgnoteの画面だけではわからない数値があり、CSVをダウンロードして手作業でスプレッドシートで集計を毎月する必要があった。また、エンタープライズ版では同じ作業を10回以上繰り返す必要があり、非常に手間がかかっていた。

### 概要
画面上で項目を選択するだけでBigQueryのSQLが実行され、データを抽出できるシステムです。従来のCSVやgnote画面では見れなかった数値が一発で出せるようになり、エンプラで10回以上必要だった作業が1回で完了するようになりました。

### 主な機能
- 画面上で項目を選択してデータ抽出
- 従来のCSVやgnote画面では見れなかった数値も一発で取得
- エンプラでの繰り返し作業を1回に集約
- BigQueryからの柔軟なデータ取得

### 使用技術
Python / Google BigQuery / Cloud Run

### 対象ユーザー
データ分析したい全スタッフ

## 8. gnoteコール入力支援・優先度判定

**フォルダ名:** gnote_コール画面, ナオイ_コメント反映

### 解決する課題
クライアントの自社コールセンターで、複数台所有している顧客がいた場合に「どの車から案内すべきか」を判断するために色々なところを見る必要があり、難しいし大変だった。また、お客様の回答に応じたヒアリング内容を手動で選択・入力する必要があり、時間がかかっていた。一等書記官（WEB版車両履歴検索システム）での検索も手作業で面倒だった。

### 概要
gnoteのコール画面に便利機能を追加するChrome拡張機能セットです。複数台保有の顧客でも「どの車から案内すべきか」を自動判定。お客様の回答に応じてヒアリングすべき内容が表示され、選択するだけでgnoteのステータス・コメントが自動で反映されます。一等書記官への検索もボタン一つで実行できます。

### 主な機能

**コール優先度判定**
- 複数台保有の顧客でも「どの車から案内すべきか」を自動判定
- 車検満了日・案内すべき内容（車検か点検かなど）から優先度を判定

**回答に応じたヒアリング内容表示**
- お客様の回答に応じてヒアリングすべき内容を自動表示
- 選択するだけでgnoteのステータス・コメントが自動反映

**一等書記官連携（WEB版）**
- 顧客名・ナンバー・車台番号での検索をボタン一つで実行
- 検索結果を新しいタブで自動表示

### 使用技術
Chrome拡張機能 / JavaScript

### 対象ユーザー
クライアントの自社コールセンターオペレーター

## 9. gnoteコール・DMリスト自動反映

### 主な機能
- kintoneからgnoteへの設定を一発で反映
- RPAの実行時間を大幅短縮
- コールリスト作成画面とDMリスト作成画面で使用可能
- 管理画面と表画面のどちらからも操作可能

### 使用技術
Chrome拡張機能 / Google Apps Script / kintone API

### 対象ユーザー
営業部門、コールセンター部門

## 10. gnote入庫管理 設定支援ツール

**フォルダ名:** project

### 解決する課題

### 主な機能
- 大量の入力項目を一括で反映
- 複数店舗への同じ設定変更を一発で反映
- 変更前後の比較表示で確認しやすい
- 安全設計：選択は自動、登録は人間が実行

### 使用技術
Chrome拡張機能 / Google Apps Script / Google Sheets

### 対象ユーザー
店舗管理部門、システム管理部門

## 13. コールロープレAI訓練システム

**フォルダ名:** コールセンター_コールロープレ

### 解決する課題
新人採用を毎月行っており、そのための研修も毎月何時間も使っていた。研修担当者の負担が大きく、新人が習得するまで効率的に練習できる環境がなかった。

### 概要
AIが「顧客役」となってスタッフと会話練習（ロールプレイ）ができるシステムです。マイクで話しかけるとAI顧客が応答し、終了後に「良かった点」「改善点」を自動でフィードバック。習得するまで何度でも自動で練習できます。

### 主な機能
- AIとの音声会話による営業練習
- 5段階の難易度（初心者〜上級者）
- 80種類以上のシナリオ（様々な顧客タイプ）
- AI自動採点（導入トーク、切り返し、クロージング等）
- 「良かった点」「改善点」の自動フィードバック
- スタッフ別の進捗・成績管理

### 使用技術
Next.js / OpenAI（Whisper, TTS, GPT） / kintone

### 対象ユーザー
コールセンタースタッフ、研修担当者、マネージャー

## 14. ふるさと納税レビュー収集・分析・返信AI生成

**フォルダ名:** 事務局_加西市PJ, 事務局_レビュー返信

### 解決する課題
Amazon、Yahoo!ショッピング、さとふるなど複数のECサイトで約1万件近いページにレビューが散在しており、人力での確認は不可能だった。レビューの存在を把握できず、悪いレビューにも対応できていなかった。

### 概要
複数のECサイトからレビューを自動で収集し、1万件以上のレビューを分析するシステムです。何が良くて何が悪かったのかを可視化し、今後の提案にも活用。悪いレビューにも即座に対応でき、AIによる返信案の自動生成機能も搭載しています。

### 主な機能
- Amazon、Yahoo!ショッピング、さとふる等からのレビュー自動取得
- 1万件以上のレビューを分析・可視化
- 何が良くて何が悪かったのかを把握、今後の提案にも活用
- レビュー内容をAI（ChatGPT）が分析し返信案を自動生成
- 毎日定時に自動実行（スケジュール設定可能）

### 使用技術
Google Apps Script / Playwright / OpenAI GPT API / Google Sheets

### 対象ユーザー
事務局（ふるさと納税事業担当）

## 15. 請求書AIダブルチェック

**フォルダ名:** 請求書ダブルチェック

### 解決する課題
請求書PDF/Excelの内容を目視でチェックするのに時間がかかっていた。丁寧に確認してくれていたが、件数が多く負担が大きかった。

### 概要
kintoneから請求書PDFを自動取得し、Dify（ワークフロー）とOpenAIのLLMを使って金額・クライアント名・商品内容などを複数の観点から自動検証するシステムです。チェック結果（OK/NG）がスプレッドシートに自動記録され、問題があれば即座に把握できます。

### 主な機能
- kintoneから請求書PDF/Excelを自動ダウンロード
- AIによる複数観点での自動検証（金額、クライアント名、内容など）
- チェック結果をスプレッドシートに自動記録
- エラー検出時は再チェック対象として自動マーク

### 使用技術
Google Apps Script / Dify（ワークフロー） / OpenAI LLM / kintone API / Google Sheets

### 対象ユーザー
事務局（請求書管理・経理部門）
